# Система учета заказов и клиентов

Система учета заказов и клиентов — это инструмент для управления информацией о клиентах и их заказах.
Проект предоставляет удобные механизмы для просмотра, добавления и фильтрации данных, а также формирования отчетов за разные периоды времени.

## Мотивация проекта

Основной мотивацией для создания данного проекта была необходимость автоматизации рутинных задач, связанных с управлением заказами и клиентами. Мы хотели разработать решение, которое позволит компаниям эффективно вести учет, анализировать данные и улучшать взаимодействие с клиентами.

## Проблема, которую решает проект

Проект решает проблему неэффективного ведения учета заказов и клиентов вручную. Он упрощает процессы поиска информации, формирования отчетов и анализа данных, что значительно экономит время и ресурсы компании.

## Отличительные особенности проекта

Наш проект выделяется следующими особенностями:
Удобный интерфейс для быстрого поиска и фильтрации данных.
Гибкая система отчетности, позволяющая  отчеты за временные интервалы.
Подробная информация о каждом клиенте, включая историю заказов и контакты.

## Доступные функции

1. **Просмотр заказов** и **клиентов**

**Быстрая навигация:** Пользователи могут мгновенно переходить между списками заказов и клиентов благодаря удобной панели навигации.

**Подробная  информация:** Каждый заказ и клиент содержат подробную информацию, такую как  контактные данные, статусы заказов и много другое. 

**История изменений:** Возможность отслеживания всех изменений, внесенных в заказы и профили клиентов, с указанием даты и времени каждого изменения.

2. **Добавление заказов и клиентов и особенности**

*Возможность добавления множества товаров в один заказ*

*Возможность редактировать **клиенты** и **заказы***

3. **Интеграция с другими системами**

**API-интерфейсы:** Возможность интеграции с другими бизнес-приложениями через API, что позволяет автоматизировать обмен данными и синхронизацию информации.

**Поддержка популярных платформ:** Готовая интеграция с популярными CRM-системами, бухгалтерскими программами и складскими системами для упрощения процесса внедрения.

4. **Отчеты за год, месяц, неделю**
**Гибкость настроек:** Пользователь может выбирать период времени для генерации отчета.
**Экспорт данных:** Пользователь может экспортировать отчет в личных целях.

5. **Фильтры и поиск**

**Сортировка:** Пользователь может отсортировать данные под себя и в случае если его не устроит очистить все фильтры.
**Поиск:** Возможность поиска по всем полям таблицы.

**ВАЖНО**

**Описание API для OrderAndCustomerAccountingSystem**

Базовый URL: api/
**1. Получение JWT токена**

URL: /api/token/

Метод: POST

Описание: Получение JWT токена для авторизации.

Тело запроса:
```
{
    "username": "admin",
    "password": "admin"
}
```

Ответ:

```
{
    "access": "<ACCESS_TOKEN>",
    "refresh": "<REFRESH_TOKEN>"
}
```

Примечание:
Добавьте access токен в заголовок запросов:
Authorization: Bearer <ACCESS_TOKEN>

**2. Обновление JWT токена**

URL: /api/token/refresh/

Метод: POST

Описание: Обновление Access токена с использованием Refresh токена.

Тело запроса:

{
    "refresh": "<REFRESH_TOKEN>"
}

Ответ:

{
    "access": "<NEW_ACCESS_TOKEN>"
}

**3. Работа с клиентами**

3.1 Добавление клиента

URL: /api/clients/add/

Метод: POST

Описание: Добавляет нового клиента.

Тело запроса:

{
    "first_name": "Иван",
    "last_name": "Иванов",
    "middle_name": "Иванович",
    "mobile_phone": "+79001234567",
    "email": "ivanov@example.com"
}

Ответ:

{
    "success": true
}

3.2 Получение одного клиента по ID

URL: /api/clients/get/<int:id>/

Метод: POST

Описание: Возвращает информацию о клиенте по его ID.

Ответ:

{
    "client": {
        "id": 1,
        "first_name": "Иван",
        "last_name": "Иванов",
        "middle_name": "Иванович",
        "mobile_phone": "+79001234567",
        "email": "ivanov@example.com"
    }
}

3.3 Получение списка клиентов

URL: /api/clients/get/

Метод: POST

Описание: Получает список клиентов с ID, начиная с start_id.

Тело запроса:

{
    "start_id": 0
}

Ответ:

{
    "clients": [
        {
            "id": 1,
            "first_name": "Иван",
            "last_name": "Иванов",
            "email": "ivanov@example.com"
        }
    ]
}

3.4 Редактирование клиента

URL: /api/clients/edit/<int:id>/

Метод: POST

Описание: Редактирует данные клиента по его ID.

Тело запроса:

{
    "first_name": "Петр",
    "last_name": "Петров"
}

Ответ:

{
    "success": true
}

3.5 Удаление клиента

URL: /api/clients/delete/<int:id>/

Метод: DELETE

Описание: Удаляет клиента по его ID.

Ответ:

{
    "success": true
}

**4. Работа с заказами**

4.1 Добавление заказа

URL: /api/orders/add/

Метод: POST

Описание: Добавляет новый заказ для клиента.

Тело запроса:

{
    "client": 1,
    "product": "Ноутбук",
    "quantity": 2,
    "price": "1500.00",
    "description": "Заказ ноутбука"
}

Ответ:

{
    "success": true
}

4.2 Получение одного заказа

URL: /api/orders/get/<int:id>/

Метод: POST

Описание: Возвращает информацию о заказе по его ID.

Ответ:

{

    "total_price": 3000.00,
    "description": "Заказ ноутбука",
    "status": "Pending",
    "created_at": "17.12.2024 21:00"
}
